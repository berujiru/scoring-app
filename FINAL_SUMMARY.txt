## 🚀 How It Works

### Events List Page (`/events`)
```
1. User navigates to /events
2. useEvents hook fetches all events from API
3. Events displayed in responsive grid (1/2/3 columns)
4. Each event shown in EventCard component
5. User can search → results filter in real-time
6. User can paginate → shows 6 events per page
7. User clicks event → navigates to /events/:id
```

### Event Detail Page (`/events/:id`)
```
1. User clicks event card
2. EventDetail page loads event data
3. Shows event header, stats, and related data
4. Displays contestants, judges, criteria
5. User can click back button to return to list
6. All data loads with proper loading/error states
```

### Search & Pagination
```
Search:
  • Real-time as user types
  • Filters by event name OR description
  • Case-insensitive
  • Pagination resets to page 1

Pagination:
  • 6 items per page (configurable)
  • Smart page display (shows 1...5...10)
  • Previous/Next + page numbers
  • Auto-hides if only 1 page
```

## 🎯 DRY Principle Applied

### Reusable Components
```typescript
// EventCard - used for EACH event in grid
// Pagination - used on any paginated list
// useEvents - logic reused wherever needed

Benefits:
✅ Single source of truth for each feature
✅ Easy to update all at once
✅ Consistent styling and behavior
✅ Less code duplication (~60% reuse)
✅ Easier testing and maintenance
```

### Component Reuse Example
```
EventCard is rendered 6 times per page in grid
- Mobile: 6 cards visible (1 per row)
- Tablet: 6 cards visible (2 per row)
- Desktop: 6 cards visible (3 per row)

All using THE SAME EventCard component
= No code duplication
= Easy to update all cards at once
```
